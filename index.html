  • 2º Dados de saída

    • geralmente se refere a informações que um sistema, programa ou processo produz como resultado de uma operação ou função específica.
    • Esses dados são exibidos na interface do usuário, armazenados em arquivos, podem ser enviados para outros sistemas ou usados de várias maneiras.
    • A natureza dos dados de saída depende do contexto do sistema ou da aplicação específica.
    • Salvamento de dados em arquivos, como documentos, planilhas, ou formatos específicos de dados.

  • 4º Tipos de metodologias

    • Existem diversas metodologias aplicadas em diferentes áreas, desde o desenvolvimento de software até a gestão de projetos e processos.
    • Flexibilidade, colaboração, entrega incremental, adaptação a mudanças. - **Exemplos:** Scrum, Kanban, XP (Extreme Programming).
    • Abordagem sequencial, fases distintas, ênfase na documentação. - **Aplicação:** Desenvolvimento de software, engenharia de sistemas.

  • 5º Tratamento da estrutura de dados

    • A escolha da estrutura de dados certa é crucial.
    • Por exemplo, se você precisar de acesso rápido aos elementos por índice, um array pode ser uma escolha certa.
    • Para evitar a manipulação do desempenho ao inserir ou remover elementos, é importante escolher a estrutura de dados correta.
